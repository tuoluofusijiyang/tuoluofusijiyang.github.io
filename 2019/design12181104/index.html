阿三：飞哥，我这里使用的设计模式 - 命令模式，
将动作执行（LampOnCommand，TvOnCommand……）与接收者（Lamp，Tv）包装到对象里面，对外暴露的只有一个 Command 接口中的 execute 方法，其他对象不需要知道那个接收者执行了什么动作，只需要知道调用 execute，就能完成一个请求的操作，这个对象，与其他对象没有关联，完全解耦和，如果需要做新增，不需要修改原有代码，拓展接收者类和动作执行类，就能实现功能。
